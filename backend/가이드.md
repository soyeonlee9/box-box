# 🚀 백엔드 초기 세팅 및 작업 순서 가이드

현재 `backend` 폴더가 비어있습니다. 프론트엔드 개발 환경 구성이 완료되었으므로, 프론트엔드와 통신할 API 서버를 구축해야 합니다.

가장 보편적으로 많이 사용되는 **Node.js + Express** 스택을 기준으로 기본적인 백엔드 작업 순서를 정리했습니다. (Python FastAPI, Spring Boot 등 다른 스택을 사용할 예정이시라면 말씀해 주세요!)

---

## 🏗️ 1단계: 프로젝트 초기화
터미널을 열고 `backend` 폴더로 이동하여 Node.js 프로젝트를 초기화합니다.

```bash
# 백엔드 폴더로 이동 (현재 프론트엔드 폴더에 있다면 한 단계 위로 이동 후 접근)
cd ../backend

# package.json 생성
npm init -y
```

## 📦 2단계: 필수 패키지 설치
API 서버 구동에 필요한 기본 프레임워크와 유틸리티 라이브러리를 설치합니다.

```bash
# 기본 프레임워크 및 필수 라이브러리 설치
npm install express cors dotenv

# 개발 편의를 위한 자동 재시작 도구 설치 (개발용)
npm install -D nodemon
```
* `express`: 웹 서버 프레임워크
* `cors`: 프론트엔드와 백엔드의 포트가 다를 때 발생하는 접근 권한(CORS) 오류 해결
* `dotenv`: 포트 번호, DB 비밀번호 등 환경 변수 관리
* `nodemon`: 코드를 수정할 때마다 서버를 자동으로 재시작해주는 도구

## 📂 3단계: 기본 폴더 구조 생성
유지보수가 쉽도록 기능별로 폴더를 나눕니다. `backend` 폴더 아래에 다음과 같이 디렉토리를 생성합니다.

```text
backend/
├── src/
│   ├── controllers/   # 핵심 비즈니스 로직 및 API 동작 처리
│   ├── routes/        # URL 경로(Endpoint) 설정 및 라우팅
│   ├── models/        # 데이터베이스 스키마 및 DB 접근 로직
│   ├── config/        # DB 연결 및 환경 설정 코드
│   └── app.js         # Express 앱 초기화 및 미들웨어 설정 파일
├── .env               # 환경 변수 파일 (DB 정보, 포트 번호 등 - Git 업로드 X)
├── .gitignore         # Git에 올리지 않을 파일 목록
├── server.js          # 서버 실행 엔트리 파일
└── package.json
```

## 💻 4단계: 기본 서버 코드 작성
가장 먼저 실행될 서버 파일들을 작성합니다.

`server.js` (서버 실행 진입점):
```javascript
require('dotenv').config();
const app = require('./src/app');

const PORT = process.env.PORT || 5000;

app.listen(PORT, () => {
  console.log(`✅ Server is running on port ${PORT}`);
});
```

`src/app.js` (Express 앱 및 미들웨어 설정):
```javascript
const express = require('express');
const cors = require('cors');

const app = express();

// 미들웨어 설정
app.use(cors()); // 프론트엔드 요청 허용
app.use(express.json()); // JSON 형태의 요청 본문 처리

// 테스트 API 라우터
app.get('/api/test', (req, res) => {
  res.json({ message: "프론트엔드와 백엔드가 성공적으로 연결되었습니다! 🎉" });
});

module.exports = app;
```

## ⚙️ 5단계: 실행 스크립트 추가
`backend/package.json` 파일을 열어 `scripts` 부분에 서버 실행 명령어를 추가합니다.

```json
"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js"
}
```

## 🚀 6단계: 서버 실행 및 프론트엔드 연동 테스트
터미널에서 서버를 실행합니다.
```bash
npm run dev
```
이제 프론트엔드 코드에서 `fetch('http://localhost:5000/api/test')` 경로로 요청을 보내 백엔드와 정상적으로 연동이 되는지 테스트를 진행하면 됩니다.
